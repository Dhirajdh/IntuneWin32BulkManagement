# Security Policy

## Reporting a vulnerability
If you discover a security vulnerability in **IntuneWin32BulkManagement**, please help us keep the project safe by responsibly disclosing it.

1. **Do not** publish any details publicly.
2. Email the details directly to **security@yourdomain.com** (replace with your actual contact address).
3. Include:
   - A clear description of the vulnerability.
   - Steps to reproduce.
   - Potential impact.
   - Any suggested mitigations or fixes.
4. We will acknowledge receipt within 48 hours and aim to provide a fix or mitigation as quickly as possible.

## Preferred disclosure timeline
- **Critical** (remote code execution, credential leakage, etc.): Immediate private disclosure, with a fix released within 7 days if feasible.
- **High/Medium** (privilege escalation, data exposure): Private disclosure, fix within 14‑30 days.
- **Low** (information disclosure, minor bugs): Private disclosure, fix at the maintainers' discretion.

Thank you for helping make this project more secure.
